# systemd

Шаблон для мониторинга состояния юнитов systemd Linux через Zabbix agent 2 для Zabbix 4.4.

Контроль состояния сервисов будет выполняться для всех сервисов, удовлетворяющих следующим условиям:

- сервис находится в состоянии active/enabled;

- имя сервиса не удовлетворяет следующему регулярному выражению –

`^(anacron.*|systemd-fsck.*|quotarpc.*|NetworkManager.*|cpupower-gui.*|e2scrub_reap.*|systemd-pstore.*|auditd.*|grub-common.*|avahi.*)`

,заданному в макросе `{$SYSTEMD.NAME.SERVICE.NOT_MATCHES}` на уровне шаблона.

Для задания более узкого количества контролируемых сервисов, можно использовать макрос (создаваемый на уровне хоста)

`{$SYSTEMD.NAME.SERVICE.MATCHES}` равный по значению регулярному выражению, соответствующему именам интересующих сервисов.
